# Workflow — manage-study-list

활성화 조건은 `agent/skills/manage-study-list/SKILL.md` 참조.

---

## 형식 표준

```markdown
# TODO — 연구/학습 주제

## [주제 카테고리 1]

### [구체 주제]

**배경**: 이 주제가 왜 떠올랐는지

**연구 질문**:
- 질문 1
- 질문 2

**관련 문서**:
- [[관련 neuron 문서]]
- 외부 자료 경로/URL

**우선순위**: 높음 / 중간 / 낮음 (선택)
**상태**: 대기 / 진행중 / 완료

---
```

## 표준 카테고리 (확장 가능)

- **연구 주제** — 깊이 있는 학습/연구
- **읽을 자료** — PDF, 논문, 책
- **실험/프로젝트 아이디어** — 시도해볼 것
- **문서화 필요** — neuron에 정리할 것
- **재검토 필요** — 기존 지식 중 업데이트 필요

## Step 1: 의도 파악

| 사용자 요청 | 동작 |
|----------|------|
| "추가" / "공부할 것" | 항목 추가 |
| "보여줘" / "현재 todo" | 조회 |
| "완료" / "진행중" / "보류" | 상태 갱신 |
| "정리" / "아카이브" | 재구성 |
| "다음 추천" / "공백" | neuron/ 분석 기반 제안 |

---

## Step 2: 현재 todo.md 읽기

기존 카테고리·구조 파악. 신규 카테고리 추가 가능하지만 우선 기존 활용.

---

## Step 3: 작업 수행

### 3.1 추가

- 적절한 카테고리 선택 (없으면 사용자에 확인 후 신규)
- 형식 표준대로 항목 추가 (Edit, 정확한 위치에 삽입)
- 관련 neuron/ 문서 자동 링크 (Grep으로 탐색)

### 3.2 조회

- 전체: 카테고리별 카운트 + 상위 우선순위
- 검색: 키워드 매칭 항목

### 3.3 상태 갱신

- Edit으로 해당 항목의 **상태** 라인 변경
- "완료" 항목은 별도 `todo_archive.md`로 이동 옵션

### 3.4 정리

- 카테고리 재배치 (의미 단위로)
- 중복 제거 (사용자 확인)
- 완료 항목 아카이브

### 3.5 neuron/ 분석 기반 제안

- `find neuron/ -type d` → 카테고리 분포 파악
- 문서 수 적은 leaf, 외부 참조는 많은데 본문 적은 영역 식별
- 사용자에 제안:
```
neuron/ 분석 결과 공백 영역:
- compiler/ai-compiler/iree: 문서 0개, but 외부 참조 多
- compiler/ssa: 문서 1개 (다른 곳에서 [[ssa]] 5회 참조)

todo에 추가할까요?
```
